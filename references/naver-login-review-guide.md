## 1. 이번에 막혔던 핵심 문제와 원인

### 문제 A — "사업자등록증 제출 요청"으로 반려됨 (2026.06.08)
- 원인: API 설정에서 **휴대전화번호 조회 권한**을 선택했기 때문.
- 네이버는 휴대전화번호처럼 민감한 정보를 요구하면 "정상 사업자인지" 확인하려고 사업자등록증을 요구함.
- 이용자 식별자(익명 ID)만 받으면 사업자등록증은 요구하지 않음.

### 문제 B — "제공 정보 활용처 확인"이 계속 빨간색
- 원인: 이용자 식별자 외에 추가 정보(이름/이메일/성별/생일/출생연도 등)를 선택하면,
  그 정보가 서비스에서 활용되는 화면 캡처를 올려야 함.
- 네이버 로그인 동의창 캡처는 **활용 사례로 인정 안 됨 → 오히려 반려 사유** (공식 명시).

### 문제 C — "재검수 요청" 버튼이 계속 비활성(회색)
- 원인: 네이버는 **"네이버 로그인 검수 상태" 탭 안에서 실제로 저장되는 변경**이 있어야 버튼을 켜줌.
- API 설정 탭에서 권한을 바꾼 것만으로는 검수 상태 탭이 "변경 없음"으로 인식해 버튼이 안 켜짐.

## 2. 최종 해결책 (A안 — 이용자 식별자만 남기기)

### 2-1. API 설정 탭
- 경로: 네이버 개발자센터 > 내 애플리케이션 > 휴먼프리즘 > **API 설정** 탭
- "네이버 로그인 > 제공 정보 선택"의 모든 체크박스(필수/추가) 해제:
  회원이름, 연락처 이메일 주소, 별명, 프로필 사진, 성별, 생일, 연령대, 출생연도, 휴대전화번호 → **전부 해제**
- 이용자 식별자(동일인 식별 정보)는 기본 제공이라 그대로 둠.
- 맨 아래 **"수정"(저장)** 버튼 클릭.
- 결과: "제공 정보 활용처 확인" 항목이 검수 상태 탭에서 통째로 사라짐(빨간색·캡처 요구 제거), 사업자등록증 요구도 사라짐.

### 2-2. 네이버 로그인 검수 상태 탭
- 경로: 같은 화면에서 **"네이버 로그인 검수 상태"** 탭으로 이동.
- "서비스 적용 형태 확인" 섹션의 체크박스 중 **"기타" 체크박스를 체크**하고, 옆 텍스트칸에 글자 입력.
  - 주의: "기타" 칸은 메커니즘이 특이함 — 칸에 커서를 넣으면 자동 체크되며 글자 색이 초록으로 바뀜.
  - 마침표(.) 하나만으로는 변경 인식이 안 될 수 있음. **실제 글자**를 넣어야 안전. (이번엔 "이용" 정도 입력으로도 됨)
  - 위쪽 3개 체크박스(신규 회원가입 적용 / 단순 로그인 인증 적용 / 게시판 일부 메뉴 적용)는 원래 체크 상태이므로 건드리지 말 것.
- **"소명 내용 입력"** 칸에 입력 (개인정보·사업자번호는 넣지 말 것):
  ```
  네이버 로그인을 이용자 식별 용도로만 사용하며 추가 개인정보는 조회하지 않습니다. 검수 승인 부탁드립니다.
  ```
- **파일 첨부: 비워둠.** 사업자등록증 첨부하지 말 것 (권한 회수했으므로 불필요, 오히려 검토 포인트만 늘림).
- "네이버 로그인 이용 절차 확인용 캡처 파일 5개"는 **그대로 둠** (로그인 동작 과정 자료라 필요).
- "추가 제출 서류 확인" = **"없음"** 선택 유지.
- 맨 아래 **"재검수 요청"** 버튼이 초록색으로 활성화되면 클릭.

## 3. 결과 확인
- 클릭 후 상단 상태가 "승인거부" → **"검수요청"**으로 변경되면 정상 접수.
- 결과는 신청일로부터 2~3 영업일 내 **네이버 계정 등록 이메일**로 통보.
- 승인되면 "네이버 로그인 검수 상태" 탭 자체가 사라짐.

## 4. 트레이드오프 / 후속 메모
- 네이버 로그인에서 이름·이메일·생년월일을 받아오지 않음.
  → 우리 서비스는 사주 정보(생년월일·성별·시간)를 사용자가 만세력 입력 폼에 직접 입력하므로 운영에 지장 없음.
  → 이름/이메일이 필요하면 가입 후 우리 자체 화면에서 따로 입력받으면 됨.
- 만약 나중에 네이버에서 프로필 정보를 받아오고 싶어지면, API 설정에서 해당 항목을 다시 체크하고
  human-prism.com의 회원가입/내정보 페이지(해당 항목이 보이는 화면) 캡처를 올린 뒤 재검수해야 함.
  (절대 네이버 동의창 캡처를 올리지 말 것 → 반려.)
- 카카오 로그인은 이미 새 도메인으로 Redirect URI 갱신 및 검증 완료됨.
